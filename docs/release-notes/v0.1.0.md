# v0.1.0

First public release of **MAVLink Generator** — desktop UI and CLI for generating MAVLink dialect bindings from XML.

## Highlights

- **Desktop app** (Windows, macOS, Linux) — browse for XML and output, select languages, validate and generate with a live log
- **CLI** — same generator for scripts and CI (`mavlink-generator validate`, `generate`, `list-languages`)
- **8 target languages** — Dart, Python, C, C++, JavaScript, TypeScript, C#, Rust
- **Outputs per language** — dialect files, runtime helpers, and examples under `generated/<lang>/`

## Quick start (desktop)

1. Install the `.msi` (Windows) or platform installer from [Releases](https://github.com/HatemslaH/mavlink_generator/releases/tag/v0.1.0).
2. Open **MAVLink Generator**.
3. Set XML to your dialect (default points at `rt_rc.xml` when run from a dev tree; pick the file manually in the installed app).
4. Choose output folder and languages → **Validate**, then **Generate**.

## Quick start (CLI)

```bash
mavlink-generator validate path/to/rt_rc.xml
mavlink-generator generate --lang rust --output ./generated
```

## Known limitations

- macOS build is **Apple Silicon only** (no Intel `.dmg` in this release).
- Windows installer is **unsigned** — SmartScreen may show a warning on first run.
- XML definitions are not bundled yet; point the app at your `mavlink/message_definitions` path or use **Browse**.
